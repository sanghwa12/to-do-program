# 개인용 할 일 관리 앱 (Personal Task Manager)

> 이 파일은 Claude Code가 세션 시작 시 자동으로 읽는 프로젝트 컨텍스트입니다.
> 개발자는 프로그래밍 **입문자**(코딩 지식 거의 없음)입니다.
> 코드는 명확하게 쓰고 한글 주석을 충분히 달아주세요. 설명은 비유를 섞어 쉽게.

## 이 앱의 정체 (한 문장)
**계획형 인간이 아닌 사용자가, 일할 때 생각나는 할 일을 부담 없이 쏟아부으면,
앱이 날짜·우선순위·카테고리로 정리해서 보여주는 컴퓨터용 할 일 관리 프로그램.**

> 사용자의 표현: "할 일 관리라기보다 **나만의 행정 도우미**" —
> 할 일 + 공지(알아둘 것) + 달력 + 백업으로 자라는 중 (2026-07-03).

## 핵심 사용 시나리오
1. 일하다가 할 일이 떠오름 → 앱에 **한 줄 입력하고 바로 일로 복귀** (입력이 3초 안에 끝나야 함)
2. 필요할 때 앱을 열면 → **정리된 목록**(오늘 할 것 / 급한 것 / 종류별)이 보임
3. 끝낸 일은 체크 → 목록에서 정리됨

## 핵심 제약 / 원칙 (반드시 준수)
- 개발자는 입문자: 한 번에 하나의 마일스톤씩, 실행해서 눈으로 확인 가능한 최소 단위로 진행.
- **컴퓨터 전용.** 폰 동기화·알림·이메일은 범위 밖 (필요해지면 그때 논의).
- 백엔드 서버를 만들지 않는다. Firebase/Supabase 등 클라우드 백엔드도 사용하지 않는다.
- 비용 0원 유지 (개인용, 사용자 1명).
- 데이터는 브라우저 로컬 **IndexedDB**에 저장 (localStorage/sessionStorage 금지).
- API 키·시크릿을 코드에 하드코딩하지 않는다.
- **입력 마찰 최소화가 최우선 가치.** 기능을 추가할 때 "입력이 느려지거나 복잡해지는가?"를 항상 확인.

## 기술 스택
- Frontend: React + Vite
- 로컬 저장: IndexedDB (`Dexie.js` 사용 — 순수 IndexedDB보다 문법이 쉬워 입문자 친화적)
- (나중에, 선택) PWA로 설치형 앱처럼 만들기

## 데이터 모델 (Task)
```ts
type Task = {
  id: string;                          // 자동 생성 (crypto.randomUUID)
  title: string;                       // 할 일 내용 (필수)
  memo?: string;                       // 상세 메모 (선택)
  dueDate?: string;                    // 날짜, "YYYY-MM-DD" (마감일·당일·기간의 끝. 없으면 날짜 미정)
  startDate?: string;                  // 기간의 시작일 (dateKind가 "range"일 때만)
  dateKind?: "due" | "day" | "range";  // 날짜의 의미: ~까지 마감 | 그날 당일 | 기간
  priority?: "low" | "med" | "high";   // 우선순위 (선택)
  repeat?: "daily" | "weekdays" | "weekly" | "monthly" | "yearly" | "monthlyNth"; // 반복 주기 (weekdays=주중 매일)
  repeatNth?: 1 | 2 | 3 | 4 | "last";  // monthlyNth일 때: 몇째 주 (매월 N번째 요일)
  repeatWeekday?: number;              // monthlyNth일 때: 요일 (0=일 ~ 6=토)
  category?: string;                   // 카테고리 (예: "업무", "개인") (선택)
  done: boolean;                       // 완료 여부
  createdAt: string;                   // 생성 시각 (ISO 8601, 자동)
  deletedAt?: string | null;           // 휴지통에 버린 시각 (있으면 휴지통에 있음, 없으면 목록에)
};
```
- 입력할 때 title만 필수. 나머지는 **나중에 붙여도 되는 선택 사항** (쏟아붓기를 막지 않기 위해).

## 빌드 순서 (마일스톤 — 하나씩 완성)
1. **빠른 입력 + 기본 목록** — React 뼈대, IndexedDB 저장, 한 줄 입력 → 목록 표시 → 완료/삭제/수정
2. **정리 뷰** — 오늘 할 일 / 날짜별 / 우선순위 / 카테고리로 나눠 보기. 날짜·우선순위·카테고리 입력 UI 추가
3. **파일 내보내기 (백업)** — 전체 할 일을 Obsidian 호환 마크다운(.md)으로 다운로드.
   데이터가 브라우저에만 갇히지 않게 하는 안전장치
4. **정리본(리뷰) 화면** — 형태 미정 (예: 주간 돌아보기). 사용자가 M1~2를 써본 뒤 결정
5. (선택) PWA 설치, UI 다듬기
6. **알아둘 것** — 할 일이 아닌 정보 (📢 일정 공지 + 📎 참고). 오늘 탭 상단 게시판 + 달력 표시
7. **반복 할 일** — 매일/매주/매달/매년. 완료하면 기록 남기고 다음 회차로

현재 상태: **마일스톤 1·2·3 완료. 여러 줄 가져오기·삭제 안전장치 추가. 스티커 메모 이전 진행 중.**
- 마일스톤 4(정리본): 사용 경험 쌓은 뒤 형태 결정 (미착수).
- 마일스톤 5(PWA): 개발 중 캐시 문제로 임시 selfDestroying(오프라인 꺼둠). 최종에 재설치·재검증.
- 개발/테스트는 `npm run dev`(localhost:5173)에서. 자세한 경위는 `docs/작업노트.md`.

## 미해결 / 나중에 결정
- "정리본"의 구체적 형태 (마일스톤 4) — 사용자가 앱을 써본 뒤 결정.
- 백업 가져오기(import/복원) 기능 — 필요해지면 결정.
- UI 디자인 방향.
- PWA 설치형으로 만들지 여부.
- Obsidian **실시간 연동**(vault를 저장소로 사용)은 검토 후 **하지 않기로 결정** (2026-07-02).
  이유: 브라우저의 파일 접근 제약 + 입문자 난이도. 대신 마크다운 내보내기(F06)로 호환.
