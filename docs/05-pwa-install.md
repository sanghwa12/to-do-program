# F05 · PWA 설치 (선택)

## 목적
브라우저 주소를 치는 대신, 바탕화면 아이콘으로 여는 **설치된 프로그램처럼** 만들기.
인터넷 없이도 앱이 열리게 한다. (데이터는 원래부터 IndexedDB라 오프라인 OK — 이건 앱 화면 자체의 오프라인화)

## 사용자 시나리오
- 브라우저 주소창 오른쪽의 설치 아이콘(⊕ 모양)으로 앱을 설치한다.
- 설치 후 바탕화면/시작 메뉴의 "할 일" 아이콘으로 실행 → 주소창 없는 독립 창.
- 인터넷이 끊겨도 앱이 열리고 할 일을 볼 수 있다.

## 요구사항 (검증 항목)
- [x] R1. 웹 앱 매니페스트 (`name`, `icons`, `start_url`, `display: standalone`)
- [ ] R2. 서비스 워커가 정적 파일을 사전 캐시 → 오프라인에서도 열림
      (개발 중 selfDestroying로 잠시 꺼둠. 최종 재설치 시 다시 검증 — 작업노트 참고)
- [x] R3. 설치 후 독립 창으로 실행됨 (`display: standalone`)
- [x] R4. 설치용 아이콘 192px·512px 존재

## 관련 제약 (CLAUDE.md 핵심 제약)
- 새 라이브러리 `vite-plugin-pwa` 추가 이유: 서비스 워커를 손으로 짜면 실수하기 쉬움 —
  플러그인이 빌드 시 자동 생성 (입문자 친화적).

## 관련 파일
- `vite.config.js` — VitePWA 설정 (매니페스트 내용 포함)
- `public/icon-192.png`, `public/icon-512.png` — 앱 아이콘 (파란 바탕 + 흰 체크)

## 비고 / 결정 사항
- (2026-07-02) `vite-plugin-pwa` 사용으로 결정.
  서비스 워커는 빌드 결과물에만 생성됨 → 오프라인·설치 테스트는
  `npm run build` 후 `npm run preview`(또는 배포본)에서 확인.
- (2026-07-03) **개발 중 임시로 `selfDestroying: true`** 로 전환 (오프라인 잠시 꺼짐).
  이유: 서비스 워커 캐시가 옛 코드를 붙잡아 최신 버전이 앱에 반영되지 않는 문제.
  개발은 `npm run dev`(5173)에서 진행, 앱이 안정되면 `selfDestroying`을 false로 되돌려
  오프라인을 다시 켜고 재설치. 자세한 경위는 `docs/작업노트.md` 참고.
  → 이 때문에 R2(오프라인) 검증은 최종 재설치 시점으로 미룸.
