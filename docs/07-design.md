# D00 · 디자인 원칙 · UI 단순화

> 이 문서는 특정 기능이 아니라 **화면 전체의 생김새를 관리**하는 기획서입니다.
> 기능이 늘수록 화면이 복잡해지기 쉬우니, 여기서 원칙을 정하고 주기적으로 정리합니다.

## 디자인 원칙 (이 앱의 정체에서 나옴)
1. **입력이 주인공.** 입력창이 가장 크고 눈에 띄어야 한다. 다른 요소가 입력을 방해하면 안 됨.
2. **시각적 소음 최소화.** 색·테두리·버튼을 남발하지 않는다. 기본은 무채색, 강조는 딱 필요할 때 1색.
3. **자주 쓰는 건 눈에, 드물게 쓰는 건 숨긴다.** (예: 삭제·내보내기·초기화는 평소 눈에 덜 띄게)
4. **한 화면에 한 가지 일.** 목록을 볼 땐 목록에 집중, 편집할 땐 편집만.
5. **컴퓨터 화면 기준**이되, 창을 줄여도 깨지지 않게(반응형) 유지.

## 현재 상태에서 복잡하다고 본 것들 (2026-07-03)
- 각 할 일마다 **수정·삭제 버튼이 항상 보임** → 목록이 버튼으로 빽빽함.
- **뱃지 색이 여러 종류**(마감/진행중/우선순위 3색/카테고리) → 알록달록함.
- 상단 **내보내기·가져오기**, 하단 **휴지통·모두지우기** 등 액션이 곳곳에 흩어짐.
- 탭이 **5개**(오늘/전체/날짜/우선순위/카테고리) → 많게 느껴질 수 있음.

## 단순화 후보 (무엇을 우선할지는 사용자와 확정)
- **A. 항목 버튼 숨기기**: 평소엔 제목·뱃지만, 마우스 올리거나(또는 항목 클릭 시) 수정/삭제 노출.
- **B. 색 줄이기**: 뱃지를 회색 기본으로 통일하고, 강조는 "밀림/오늘" 같은 진짜 중요한 것만 1색.
- **C. 액션 정리**: 내보내기·가져오기·초기화를 상단 "⋯ 메뉴" 하나로 모음.
- **D. 탭 줄이기**: 자주 안 쓰는 탭을 접거나 합침(예: 날짜/우선순위를 정렬 옵션으로).
