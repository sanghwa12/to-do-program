# F06 · 파일 내보내기 (백업 · Obsidian 호환)

## 목적
할 일 데이터가 브라우저(IndexedDB) 안에만 갇혀 있지 않게, **일반 텍스트 파일로 꺼낼 수 있게** 한다.
- 브라우저 데이터가 삭제돼도 백업 파일이 남는다 (데이터 소유권·안심).
- 파일은 Obsidian이 알아보는 마크다운 체크박스 형식 → Obsidian에 넣으면 그대로 체크리스트로 보임.

## 사용자 시나리오
- "내보내기" 버튼을 누르면 `.md` 파일이 다운로드된다.
- 그 파일을 메모장으로 열어도 읽을 수 있고, Obsidian 폴더에 넣으면 체크리스트로 보인다.
- 가끔(예: 주 1회) 눌러서 백업본을 만들어 둔다.

## 요구사항 (검증 항목)
- [ ] R1. "내보내기" 버튼이 있고, 누르면 전체 할 일이 담긴 `.md` 파일이 다운로드된다.
- [ ] R2. 마크다운 형식이 Obsidian 호환이다:
      미완료 `- [ ] 제목`, 완료 `- [x] 제목`. 날짜·우선순위·카테고리·메모가 있으면 같이 표기.
- [ ] R3. 파일명에 날짜가 들어간다 (예: `할일백업-2026-07-02.md`) — 백업본 구분용.
- [ ] R4. 내보내기는 저장된 데이터를 읽기만 한다 (내보내는 중 데이터 변경 ❌).
- [ ] R5. 외부 서비스 업로드 없이 순수 다운로드로 동작한다 (파일은 내 컴퓨터에만).

## 관련 제약 (CLAUDE.md 핵심 제약)
- 서버·클라우드 업로드 금지 → 브라우저의 파일 다운로드 기능만 사용.
- Obsidian과의 실시간 연동(파일 동시 편집)은 **범위 밖** — 내보내기(단방향)까지만.

## 관련 파일
- `src/export.js` — 마크다운 변환(taskToLine) + 다운로드(exportBackup)

- `src/labels.js` — 우선순위 한글 이름표 (화면·내보내기 공용)
