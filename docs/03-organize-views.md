# F03 · 정리 뷰 (오늘 · 날짜 · 우선순위 · 카테고리)

## 목적
쏟아부은 할 일을 **앱이 대신 정리해서** 보여준다.
"계획을 세우는 건 앱, 사용자는 붓기만 한다"를 실현하는 화면.

## 사용자 시나리오
- 앱을 열면 "오늘 할 일"이 먼저 보인다 — 오늘 뭐 하면 되는지 바로 파악.
- 탭(또는 버튼)을 눌러 우선순위순 / 카테고리별 / 날짜별로 바꿔 볼 수 있다.
- 쏟아부을 때 못 붙인 날짜·우선순위·카테고리를 **나중에** 각 할 일에 붙일 수 있다.

## 요구사항 (검증 항목)
- [ ] R1. 할 일에 `dueDate`(날짜), `priority`(높음/중간/낮음), `category`를
      **나중에 붙이거나 수정**할 수 있는 UI가 있다. (입력 시점 강요 ❌)
- [ ] R2. "오늘" 뷰: 날짜가 오늘이거나 지난(밀린) 미완료 할 일이 보인다.
      밀린 항목은 구분 표시(예: "N일 지남").
      날짜 종류별 규칙 (2026-07-02 추가):
      - 마감(due)·당일(day): `dueDate`가 오늘이거나 지났으면 표시
      - 기간(range): 시작일이 됐으면 표시 (진행 중이면 "진행중" 표시)
- [ ] R2b. 할 일의 날짜 뱃지가 종류를 구분해 보여준다:
      마감 `~7/2`, 당일 `7/7 당일`, 기간 `11/11 ~ 11/23`.
- [ ] R2c. **메모판(스티커)** (2026-07-03 추가): 기본(오늘) 탭 하단에
      **날짜 미정 + 미완료** 할 일 전체가 스티커 메모판 형태로 항상 보인다.
      - 노란 메모지 느낌의 판에 촘촘한 목록(여러 단)으로 — 한눈에 훑는 용도.
      - 각 줄에 체크박스: 체크하면 완료되어 판에서 바로 사라짐.
      - 우선순위 높음이 위로 (높음 → 중간 → 낮음 → 미지정), 높음은 굵게.
      - 수정·삭제 같은 세부 작업은 "전체" 탭에서 (메모판은 보기+완료 체크만 — 단순 유지).
- [ ] R3. "우선순위" 뷰: high → med → low → 미지정 순으로 정렬/그룹핑된다.
- [ ] R4. "카테고리" 뷰: 카테고리별로 묶여 보이고, 미지정 그룹도 있다.
- [ ] R4b. "카테고리" 뷰 상단에 **카테고리 필터(칩)**가 있어, 특정 카테고리 하나만
      골라 볼 수 있다. "전체"를 고르면 모든 카테고리를 묶어서 본다 (2026-07-03 추가).
      각 칩에 해당 카테고리의 할 일 개수를 표시한다.
- [ ] R4c. 특정 카테고리를 골랐을 때 **그 카테고리 전체를 한 번에 삭제**할 수 있다
      (삭제 전 확인 + 삭제 후 되돌리기). (2026-07-03 추가)
- [ ] R5. "날짜" 뷰: 날짜별로 묶여 보이고, 날짜 미정 그룹도 있다.
- [ ] R6. 뷰를 전환해도 완료/수정/삭제가 동일하게 동작한다.
- [ ] R8. **"전체" 탭 정렬 규칙** (2026-07-03 확정 — 급한 것부터, 완료는 아래):
      ① 미완료 + 날짜 있음 — 마감 빠른 순 (지난 것이 맨 위)
      ② 미완료 + 날짜 없음 — 우선순위 순 (높음→중간→낮음→없음), 같으면 최신 입력이 위
      ③ 완료 — 맨 아래
      (기존 "최신순"은 급한 일이 묻히는 문제가 있어 교체)
- [ ] R7. 뷰 로직은 저장된 데이터를 읽어 화면에서 정리한다 (별도 서버/외부 서비스 ❌).

## 관련 제약 (CLAUDE.md 핵심 제약)
- 날짜/우선순위/카테고리는 **선택 사항 유지** — 안 붙여도 앱이 동작해야 함.
- 입력창(F02)의 속도를 해치지 않는 방식으로 추가할 것.

## 관련 파일
- `src/App.jsx` — 탭 전환 + 그룹핑 로직 (groupByDate / groupByPriority / groupByCategory)
- `src/components/TaskItem.jsx` — 날짜·우선순위·카테고리 편집 UI + 뱃지 표시
- `src/date.js` — 오늘 날짜·"N일 지남" 계산

## 비고 / 결정 사항 (2026-07-02)
- 기본(첫 화면) 뷰는 "오늘". 탭 구성: 오늘 / 전체 / 날짜 / 우선순위 / 카테고리.
- 카테고리는 **자유 입력 + 이미 쓴 카테고리 자동완성(datalist)** 으로 결정
  (미리 목록을 정하는 건 비계획형 사용자에게 부담이라서).
