# F02 · 빠른 입력 + 기본 목록 (쏟아붓기)

## 목적
**이 앱의 존재 이유.** 일하다 떠오른 할 일을 3초 안에 입력하고 바로 일로 복귀할 수 있게 한다.
입력된 할 일은 목록으로 보이고, 완료 체크·수정·삭제가 가능하다.

## 사용자 시나리오
- 일하다 "아 그거 해야지" 떠오름 → 앱 입력창에 한 줄 치고 Enter → 끝. 다시 일하러 감.
- 나중에 앱을 보면 쏟아부은 할 일들이 목록으로 쌓여 있다.
- 끝낸 일은 체크박스로 완료 처리, 잘못 쓴 건 수정하거나 삭제.

## 요구사항 (검증 항목)
- [ ] R1. 화면 상단(또는 눈에 띄는 곳)에 입력창이 있고, **텍스트 입력 + Enter만으로** 할 일이 추가된다.
      (버튼 클릭 필수 ❌, 날짜·우선순위 입력 강요 ❌ — title만으로 저장)
- [ ] R2. 추가 후 입력창이 비워지고 포커스가 유지된다 (연속으로 쏟아붓기 가능).
- [ ] R3. 제목이 비어 있으면 추가되지 않는다.
- [ ] R4. 저장된 할 일 목록이 화면에 렌더링된다 (IndexedDB에서 읽어옴).
- [ ] R5. 완료 체크박스 토글 → `done` 값이 바뀌고 저장되며, 완료 항목은 시각적으로 구분된다 (취소선 등).
      - R5a. **완료 실행취소** (2026-07-03): 체크해서 완료로 바꾸면 "실행취소" 알림이 뜬다.
        오늘 탭·메모판에서는 체크 즉시 항목이 사라지므로, 잘못 눌러도 알림에서 바로 복구.
        (완료→미완료로 푸는 방향은 항목이 사라지지 않으므로 알림 없음)
- [ ] R6. 할 일의 제목/메모를 수정하고 저장하면 반영된다.
- [ ] R7. 삭제 버튼으로 지울 수 있고 IndexedDB에서도 제거된다.
      - R7a. 삭제는 곧바로 실행하지 않고 **한 번 더 확인**한다 (그 자리에서 "삭제할까요? 삭제/취소").
      - R7b. 삭제 후 **되돌리기**를 제공한다 (화면 아래 알림. 시간 제한 없이,
        되돌리기 또는 닫기(✕)를 누를 때까지 유지. 원래 그대로 복구).
      - R7c. **일괄 삭제**(전체 삭제, 카테고리별 삭제)도 확인 + 실행취소를 지원한다 (2026-07-03).
      - R7d. **휴지통**(2026-07-03): "삭제"는 진짜로 지우지 않고 휴지통으로 보낸다(`deletedAt` 기록).
        목록에선 사라지지만 데이터는 남아, 휴지통 화면에서 **복원**하거나 **완전 삭제**할 수 있다.
        "휴지통 비우기"로 전체 완전 삭제(확인 필요). 완전 삭제만 되돌릴 수 없다.
- [ ] R8. 새로고침 후에도 위 변경(추가/수정/삭제/완료)이 그대로 유지된다.
- [ ] R9. **날짜 자동 인식 — 3가지 종류** (2026-07-02 추가·확장):
      날짜 표기를 인식해 저장하고, 그 표기는 제목에서 뺀다. 표기가 없으면 날짜 없는 할 일.
      | 종류 | 표기 | 입력 예 | 저장 결과 |
      |------|------|---------|-----------|
      | 마감(due) | `~날짜` (어디든) | `도서 반납 (~7/2)` | 제목 "도서 반납" + 7/2까지 |
      | 당일(day) | 맨 날짜 (**문장 끝만**) | `연구실 안전점검 7/7` | 제목 "연구실 안전점검" + 7/7 당일 |
      | 기간(range) | `날짜~날짜` (어디든) | `학부실험 조교 11/11~11/23` | 제목 + 11/11부터 11/23까지 |
      - 비인식 예: `보고서 초안쓰기` → 날짜 없음 (날짜 강요 ❌)
- [ ] R10. 날짜가 인식되면 입력창 아래에 종류까지 표시한 미리보기를 보여줘서
      Enter 치기 전에 확인할 수 있다 (예: "📅 7/2까지 마감으로 저장됩니다").
- [ ] R12. **여러 줄 가져오기(붙여넣기)**: 여러 줄 텍스트를 붙여넣으면 줄마다 할 일로 저장한다.
      - 각 줄에 날짜 자동 인식(R9~R11)과 `#태그`→카테고리 인식을 적용한다.
      - `- [ ]`(미완료) / `- [x]`(완료) 앞머리를 인식한다 (내보내기 형식과 호환 → 왕복 가능).
      - `#`으로 시작하는 줄과 빈 줄은 무시한다 (주석·섹션 제목).
      - 넣기 전에 "N개 추가됩니다" 미리보기를 보여준다.
      - **같은 "제목+완료상태"가 이미 목록(휴지통 제외)에 있으면 건너뛰고 개수를 알려준다**
        (같은 파일 이중 가져오기 사고 방지 — 2026-07-03 추가.
        완료상태까지 비교하는 이유: 반복 할 일(F09)은 같은 제목의 완료 기록이 정상적으로
        여러 개 생기므로 — 2026-07-06 개선).
      - 백업 파일의 **공지 줄(📢로 시작)은 건너뛴다** — 공지가 할 일로 둔갑하지 않게
        (공지 복원 방법은 F08 미결정 — 2026-07-03 추가).
- [ ] R13. **우선순위 점(중요도 표시)** (2026-07-03 추가):
      - 각 할 일 제목 앞에 점으로 표시: **신호등 색** — ●높음(빨강) / ●중간(주황) / ●낮음(초록) / ○없음.
        (낮음=초록: "여유 있음"이라는 신호등 직관. 2026-07-03 파랑→초록 변경)
      - **점을 클릭하면 바로 순환 변경**: 없음 → 높음 → 중간 → 낮음 → 없음
        (수정 화면 없이 한 번 클릭으로 — 입력 마찰 최소화 원칙).
      - 우선순위 글자 뱃지는 점으로 대체해 제거 (중복 표시 방지, D00).
      - 수정 화면의 우선순위 선택은 그대로 유지 (두 방법 모두 가능).
- [ ] R11. **날짜 해석 규칙**:
      - 숫자 날짜: `7/2`, `7-2`, `2026-07-02` 형식. 연도 생략 시 올해, 이미 지난 날짜면 내년.
      - 말 날짜(마감 `~`에서만): `~오늘`, `~내일`, `~모레`, `~월요일`~`~일요일`
        (요일은 다가오는 그 요일. 오늘이 그 요일이면 오늘)
      - 당일(맨 날짜)을 문장 끝에서만 인식하는 이유: `3/4 분량 읽기` 같은
        분수 표현을 날짜로 오인하지 않기 위해서.
      - 존재하지 않는 날짜(예: `~13/40`)는 무시하고 전체를 제목으로 저장.

## 관련 제약 (CLAUDE.md 핵심 제약)
- **입력 마찰 최소화가 최우선.** 이 화면에 기능을 더할 때마다 "입력이 느려지는가?" 확인.
- 이 단계에서 날짜/우선순위/카테고리 **입력 UI는 만들지 않는다** (F03 범위). 데이터 필드만 존재.
