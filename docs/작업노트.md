## 2026-07-03 · 서비스 워커 캐시 문제 → 개발 중 self-destroying로 전환
- **문제**: 코드를 고쳐도 설치된 앱(localhost:4173)이 옛 버전에 갇혀 최신 기능이 안 보임.
  F5 새로고침도 안 먹힘. PWA 서비스 워커가 옛 화면을 캐시로 붙잡고 있어서 생긴 일.
  이 때문에 "가져오기" 기능이 있는데도 앱에는 안 보이고, 옛 코드로 가져오기가 되어
  `[x]`가 제목에 남고 `#` 안내문 줄까지 할 일로 들어가는 등 오작동.
- **결정**: `vite.config.js`의 VitePWA를 **`selfDestroying: true`** 로 변경.
  서비스 워커가 자기 자신을 지우고 캐시를 비움 → 개발 중 항상 최신 코드가 뜨게 됨.
  (오프라인 기능은 잠시 꺼짐. 앱이 안정되면 false로 되돌려 다시 켠다.)
- **개발 워크플로 확정**:
  - 개발/테스트는 **개발 서버 `npm run dev` (localhost:5173)** 에서. SW가 없어 F5가 바로 먹힘.
  - 설치형 앱(4173)은 **모든 기능이 안정된 뒤 마지막에** 다시 설치.
    데이터 이전은 우리가 만든 **내보내기 → 가져오기**로 처리 (origin이 달라 자동 이전은 안 됨).
