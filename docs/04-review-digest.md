# F04 · 하루 기록 (계획 vs 실제)

> 오래 비워뒀던 "정리본"의 형태가 사용 경험 끝에 확정됨 (2026-07-09):
> **매일 쓰는 계획서 + 일기** — 오늘의 계획을 자유롭게 쓰고, 실제 한 일과 비교한다.

## 목적
계획을 강요하는 게 아니라 **"계획 vs 실제"의 차이를 보여주는 거울.**
며칠 쌓이면 하루에 실제 소화하는 양, 계획 외 일이 치고 들어오는 정도가 보인다.

## 핵심 결정 (사용자 선택)
- **계획 = 완전 자유 작성** (할 일 앱과 독립된 줄 목록 — 일기장 느낌)
- **실제 한 일 = 그날 완료 체크한 할 일 자동 수집 + 계획 외 한 일만 수기 추가**
- **위치 = 새 "하루" 탭** (◀▶로 지난 날짜도 넘겨봄)

## 데이터 모델
```ts
type DayLog = {
  date: string;                            // "YYYY-MM-DD" — 하루당 1개 (키)
  plans: { id, text, done: boolean }[];    // 오늘의 계획 (자유 줄, 체크로 달성 표시)
  extras: { id, text }[];                  // 계획 외에 한 일 (수기)
  note?: string;                           // 회고 한 줄 (선택)
};
// Task에 completedAt?: string 추가 — 완료 체크한 시각 (해제하면 제거)
```

## 요구사항 (검증 항목)
- [ ] R1. `dayLogs` 테이블 추가 (Dexie version 3, 기존 데이터 무손실).
      Task에 `completedAt` 추가: 완료 체크 시 기록, 해제 시 제거. 반복 완료 기록에도 기록.
- [ ] R2. 새 **"하루" 탭**: 날짜별 페이지, ◀/▶ 이동 + "오늘" 복귀. 기본은 오늘.
- [ ] R3. **오늘의 계획**: 한 줄 입력+Enter로 추가 (자유 작성), 각 줄 체크(달성)·삭제.
      할 일 앱과 독립 — 계획 줄을 체크해도 할 일에는 영향 없음.
- [ ] R4. **실제 한 일**:
      ① 그날 완료한 할 일이 자동으로 보임 (completedAt이 그 날짜인 것, 완료 시각 표시)
      ② "계획 외에 한 일"을 한 줄씩 수기 추가·삭제.
- [ ] R5. **비교 요약**: "계획 N개 중 M개 달성 · 완료한 할 일 K건 · 계획 외 L건"
      + 회고 메모 한 줄 저장.
- [ ] R6. 할 일의 완료를 해제하면 자동 목록에서도 빠진다 (completedAt 제거).
- [ ] R7. 내보내기에 "## 하루 기록" 섹션 포함 (줄마다 📓 표기),
      가져오기는 📓 줄을 건너뜀 (할 일 오염 방지 — 📢·📎과 같은 규칙).

## 관련 제약 (CLAUDE.md)
- 계획 입력도 마찰 최소화: 한 줄 + Enter. 안 쓰는 날은 그냥 비어 있음 (강요 없음).
- 지난 날짜 기록도 열람·보완 가능 (일기니까).
