# docs/ — 기능 기획서 모음

이 폴더는 앱의 **각 기능마다 "무엇을, 왜, 어떻게" 만들지**를 적어두는 곳입니다.
코드를 짜기 전에 여기서 먼저 요구사항을 정하고, 짠 뒤에는 `/spec-check` 스킬로
**기획서와 실제 코드가 일치하는지** 정기적으로 점검합니다.

> 앱의 정체: **생각날 때마다 할 일을 쏟아부으면, 앱이 정리해주는 컴퓨터용 할 일 관리 프로그램.**
> (자세한 배경·제약은 프로젝트 루트의 `CLAUDE.md` 참고)

## 상태 표기 규칙

각 문서 맨 위(frontmatter)의 `상태:` 값은 다음 중 하나입니다.

- **스텁** — 뼈대만 있음. 해당 마일스톤에 착수할 때 상세화.
- **계획됨** — 요구사항은 확정됐지만 아직 코드 없음.
- **개발중** — 일부 구현됨.
- **구현완료** — 요구사항이 모두 코드로 구현되고 확인됨.

## 점검 방법

작업 중이나 커밋 전, 또는 마일스톤을 끝냈을 때 아래를 실행하세요.

```
/spec-check
```

그러면 각 기획서의 "요구사항(검증 항목)"을 실제 코드와 대조해서,
✅ 구현됨 / 🚧 부분 / ❌ 미구현 / ⚠️ 기획과 다름 을 표로 보고합니다.
