# F09 · 반복 할 일 (매일·매주·매달·매년)

## 목적
주기적으로 돌아오는 일(청소, 회의비 정산, 안전관리 대장, 기념일 등)을
한 번 등록하면 완료할 때마다 **자동으로 다음 회차가 잡히게** 한다.

## 동작 원리 (핵심 설계)
반복 할 일을 체크(완료)하면:
1. **완료 기록이 하나 남는다** — 완료된 일반 할 일 사본 ("언제 마지막으로 했지?" 확인용)
2. **원래 할 일의 날짜가 다음 회차로 넘어간다** — 목록에서 사라지지 않고 다음 날짜로
3. 실행취소하면 기록이 지워지고 날짜가 원래대로 돌아온다

## 요구사항 (검증 항목)
- [ ] R1. Task에 `repeat?: "daily" | "weekly" | "monthly" | "yearly"` 필드 추가.
      반복은 날짜(마감·당일)가 있는 할 일에만 붙는다 (기간(range)은 반복 미지원).
- [ ] R2. **입력 자동 인식**: `매일`, `매주`, `매주 월요일`, `매달`/`매월`, `매달 25일`,
      `매년`, `매년 7/21` 표기를 인식해 반복으로 저장하고 표기는 제목에서 뺀다.
      - 첫 날짜(기준일): 표기에 날짜가 있으면 그 날짜(다가오는 것),
        없으면 오늘. (예: 7/3(금)에 "안전점검 매주 월요일" → 첫 날짜 7/6 월요일)
      - 단어 경계 준수: "구매일지 작성"의 '매일'을 반복으로 오인하지 않는다.
      - 미리보기에 "🔁 매주 · 첫 날짜 ..." 표시.
- [ ] R3. **완료 동작**: 체크 시 완료 기록 생성 + 원본 날짜를 다음 회차로.
      - 다음 회차: 현재 날짜에서 주기만큼 더하되, **오늘 이전이면 오늘 이후가 될 때까지** 더함
        (밀린 반복 일을 완료하면 과거가 아닌 다음 번으로).
      - 매달은 말일 처리: 1/31 → 2/28 (그 달에 없는 날짜는 말일로).
- [ ] R4. **실행취소**: 완료 기록 삭제 + 원본 날짜 원복.
- [ ] R5. 표시: 할 일에 🔁 뱃지 (예: "🔁 매주"). 오늘 탭·달력에는 다음 회차 날짜만 표시.
- [ ] R6. 수정 화면에서 반복 주기를 선택/해제할 수 있다 (날짜 없거나 기간이면 반복 해제됨).
- [ ] R7. 내보내기에 🔁 표기 포함, 가져오기에서 🔁 표기 인식 (백업 왕복 유지).
- [ ] R8. **매월 N번째 요일** 규칙 (2026-07-06 추가, 예: 매월 첫째 주 월요일):
      - 데이터: `repeat: "monthlyNth"` + `repeatNth`(1~4 또는 "last") + `repeatWeekday`(0=일~6=토)
      - **수정 화면 드롭다운**으로 설정: 반복 종류에서 "매월 N번째 요일" 선택 →
        [첫째/둘째/셋째/넷째/마지막] × [요일] 선택. 날짜는 규칙에 따라 **자동 계산**
        (다가오는 해당 날짜. 직접 입력한 날짜는 무시됨).
      - 입력창 텍스트로도 인식: "매월 첫째주 월요일", "매달 마지막 금요일" ("주"는 생략 가능)
      - 완료 시 다음 회차 = 다음 달(또는 이후)의 해당 N번째 요일.
- [ ] R9. **주기별 세부 규칙 드롭다운** (2026-07-07 추가 — R8을 다른 주기로 확장):
      - 매주 선택 → **요일** 드롭다운 (일~토). 매달 선택 → **며칠** 드롭다운 (1~31일).
      - 날짜는 규칙에 맞는 다가오는 날로 자동 계산. 단, 이미 규칙과 맞는 날짜면 유지
        (예: 매주 월요일인데 날짜가 다다음 주 월요일로 미뤄져 있으면 그대로).
      - 매년은 날짜 입력 자체가 규칙 (매년 그 월/일). 날짜 없이 매일·매년을 고르면 오늘부터.
      - "매달 31일"은 31일이 없는 달엔 말일로 (기존 클램프 규칙과 동일).

## 비고 / 결정 (검증 반영, 2026-07-06)
- 반복 할 일의 체크박스는 **체크 표시가 남지 않고** 날짜가 다음 회차로 넘어감 (설계 의도).
  다시 체크하면 다음 회차를 미리 완료하는 것. 연타는 무시됨(중복 기록 방지 가드).
- 실행취소는 완료 후 사용자가 날짜를 직접 수정했으면 **그 수정을 존중** (날짜 원복 안 함).
- 밀린 반복을 한참 뒤에 완료해도 원래 일자 유지 (예: 1/31 밀린 매달 → 7/31. 드리프트 방지).
- 무효한 표기("매달 40일", "매년 2/30")는 조용히 강등하지 않고 통째로 무시(제목 유지).
- 기간+반복 조합("훈련 매주 7/1~7/12")은 기간으로만 저장하고 '매주'는 제목에 남김
  (지원 안 함을 사용자가 알 수 있게).

## 미결정
- "매달 31일"이 2월을 지나 저장되면 이후 기준일이 28일이 됨 (기준일 별도 저장은 필요해지면).
- 달력에 미래 회차를 미리 여러 개 보여주는 것 → 미결정 (지금은 다음 회차 하나만).
- 격주·평일만 같은 복잡한 주기 → 미결정.
