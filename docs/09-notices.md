# F08 · 알아둘 것 (일정 공지 + 참고 정보)

## 목적
할 일이 아닌 **"알아둬야 하는 것"**을 할 일과 분리해 관리한다. 두 종류가 있다 (2026-07-03 구분 확정):
- **📢 일정 공지** (날짜 있음): "정전 6/27", "외부 강연일 7/10" — 날짜가 핵심, 지나면 의미가 옅어짐
- **📎 참고 정보** (날짜 없음): "외부 강연 등록 → 윤리센터 사이트", "코어타임 10~17",
  "폐액통 꽉 채워 버리지 말 것" — **"이 상황엔 이렇게 한다"는 지식.** 낡지 않고, 필요할 때 찾아보는 것.
  개인 업무 매뉴얼에 가까움 (행정 도우미의 핵심).

**입력할 때 종류를 고르지 않는다** — 날짜를 쓰면 일정 공지, 안 쓰면 참고로 자동 분류.

## 할 일(Task)과 뭐가 다른가 (이 구분이 이 기능의 핵심)
| | 할 일 | 📢 일정 공지 | 📎 참고 정보 |
|---|---|---|---|
| 완료 개념 | 있음 (체크박스) | 없음 | 없음 |
| 날짜 | 마감/기간 | **"그 일이 있는 날"** | 없음 |
| 수명 | 완료하면 끝 | 지나면 흐려짐 | **낡지 않음** |
| 예 | 물질 정리 (~7/15) | 정전 6/27 | 외부 강연 등록은 윤리센터 |

## 데이터 모델 (별도 저장 — 할 일과 섞지 않음)
```ts
type Note = {
  id: string;        // 자동 생성
  text: string;      // 제목 (예: "외부 강연 등록")
  memo?: string;     // 상세 설명 (예: "윤리센터 사이트에서 등록") — 주로 참고 정보용
  date?: string;     // 관련 날짜 "YYYY-MM-DD" (있으면 일정 공지, 없으면 참고)
  createdAt: string; // 자동
};
```
- IndexedDB에 `notes` 테이블 추가 (Dexie version 2 — 기존 tasks 데이터는 그대로 유지됨).
- 할 일에 `kind` 플래그를 넣는 방식은 **안 함**: 모든 할 일 화면(오늘/전체/달력/메모판...)에
  "공지 빼기" 조건이 들어가 실수 여지가 커짐. 분리가 더 단순.

## 사용자 시나리오
- "공지" 판의 전용 입력줄에 `외부 강연 - 윤리센터 7/10` 치고 Enter → 날짜 자동 인식되어 등록.
- 앱을 열면 공지들이 눈에 띄는 곳에 떠 있음. 지난 공지는 흐리게.
- 공지는 체크할 게 없고, 필요 없어지면 삭제.

## 요구사항 (검증 항목)
- [ ] R1. 공지는 할 일과 분리된 `notes` 테이블에 저장 (Dexie version 2, 기존 데이터 무손실).
- [ ] R2. 공지판에 **전용 입력줄**: 텍스트+Enter로 추가, 할 일과 같은 날짜 자동 인식 재사용
      (인식된 날짜를 "그 일이 있는 날"로 저장. 기간 표기면 시작일을 사용).
      **지난 날짜도 그대로 저장** — 할 일과 달리 연도를 내년으로 올리지 않음
      (예: 7/3에 "정전 6/27" 입력 → 올해 6/27. 검증에서 잡힌 오동작 수정, 2026-07-03).
- [ ] R3. 판 표시: **"정보" 탭 맨 위** (메모와 한 탭 — 2026-07-15 위치 변경, 아래 비고).
      "📌 알아둘 것" 판 — 무채색(회색 테두리), 새 색 추가 없음 (D00).
      안에서 **📢 일정·공지 / 📎 참고** 두 구역으로 나눠 보여줌 (날짜 유무로 자동 분류).
      접기/펼치기 가능 (접으면 "📌 알아둘 것 N" 한 줄만). 접힘 상태는 세션 동안 유지.
- [ ] R3b. **오늘 날짜인 공지**는 오늘 탭 맨 위에 한 줄씩 표시 (놓침 방지, 읽기 전용).
- [ ] R4. 정렬: 일정 공지는 **임박한 순, 지난 것은 흐리게 맨 아래**.
      참고 정보는 최신순이며 **흐려지지 않음** (낡지 않는 지식).
- [ ] R5. 수정·삭제 가능. 수정에서 **내용(설명)**을 붙일 수 있다
      (예: 제목 "외부 강연 등록" + 내용 "윤리센터 사이트에서"). 삭제는 확인 + 실행취소.
- [ ] R6. 날짜 있는 공지는 **달력에도 표시** (📢 칩 — 흰 바탕+테두리로 할 일 칩과 구분).
      날짜 칸을 클릭하면 그날 공지도 목록 위에 보인다.
- [ ] R7. 내보내기(.md)에 `## 알아둘 것` 섹션으로 포함 (백업 누락 방지):
      일정 공지 `- 📢 제목 📅 날짜`, 참고 `- 📎 제목 — 내용`.
      가져오기는 📢·📎 줄을 건너뜀 (할 일 오염 방지).

## 관련 제약
- 서버 없음·IndexedDB·색 최소화 등 기존 원칙 그대로.
- 공지판이 할 일 입력창(주인공)을 밀어내지 않게 — 화면 위계 주의 (D00 원칙 1).

## 비고 / 결정 (2026-07-03 사용자 확인)
- **위치**: ~~오늘 탭 맨 위~~ → **"정보" 탭으로 이동 (2026-07-15)**.
  이유: 참고(📎)는 "오늘"과 무관한 지식이라 매일 다시 보이는 게 소음이고,
  오늘 탭이 3층(공지+할일+메모판)으로 비대해짐. 일정 공지는 달력에도 표시되므로
  오늘 탭에는 "오늘 날짜인 공지 한 줄"(R3b)만 남김.
- **달력 표시**: 한다.
- 앱의 정체성 메모: 사용자 표현으로는 "할 일 관리라기보다 **나만의 행정 도우미**" —
  이 방향(할 일 + 알아둘 것 + 달력 + 백업)으로 자라는 중.
